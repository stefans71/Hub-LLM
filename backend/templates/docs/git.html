Git & GitHub

Version control setup, GitHub integration, and how your AI engineer handles commits.

1

What is Git?

Git is a version control system that tracks every change to your code. Think of it like an unlimited undo history — you can see what changed, when, and why, and roll back to any previous state.

Your HubLLM project already has Git initialized. When you scaffolded your project, git init was run automatically.

2

Understanding Commits

A commit is a snapshot of your code at a point in time. Each commit has:

  • •A message describing what changed (e.g., "Add login page with form validation")
  • •A unique ID (a hash like a1b2c3d) so you can reference it later
  • •A diff showing exactly which lines were added, changed, or removed

Auto-commits by Claude

When Claude Code finishes a task, it automatically creates a commit with a descriptive message.

Tip: Run git log --oneline in the terminal to see a list of all commits.
3

Setting Up GitHub

GitHub is a cloud service that stores your Git repositories online. Connecting gives you cloud backup, collaboration, and a shareable link to your project.

Step 1: Create a GitHub Account

If you don't have one already, sign up at github.com/join. It's free.

Step 2: Create a Repository

  1. 1.Go to github.com/new
  2. 2.Enter a repository name (e.g., my-project)
  3. 3.Leave it as Public or choose Private
  4. 4.Do NOT initialize with a README (your project already has one)
  5. 5.Click Create repository

Step 3: Connect Your Project

In your HubLLM terminal, run these commands:

# Add your GitHub repo as a remote
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO.git

# Push your code to GitHub
git push -u origin main

After this, your code is backed up on GitHub. Every time Claude commits, you can push with git push.

4

Common Git Commands

Here are the commands you'll use most often:

# See what files have changed
git status

# See recent commits
git log --oneline

# Push commits to GitHub
git push

# Pull latest changes (if collaborating)
git pull

# See what changed in a file
git diff filename.js

# Undo uncommitted changes to a file
git checkout -- filename.js
5

Branching (Advanced)

Branches let you work on features in isolation without affecting your main code.

# Create a new branch
git checkout -b my-feature

# Switch back to main
git checkout main

# Merge your feature into main
git merge my-feature
Tip: For most projects, working directly on main is fine. Branching becomes useful when collaborating with others.
