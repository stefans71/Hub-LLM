Workspace Guide

A detailed walkthrough of every panel in your HubLLM workspace.

1

Overview

The HubLLM workspace is your all-in-one AI development environment. It's organized into five main areas, each designed to keep you productive without switching between tools.

Top Bar Model selector · VPS status · Project settings
File Explorer Project tree
Terminal Chat Claude Code
Preview Panel Live preview
LLM-Dev Panel Terminals · Editor · Files · Docker · Logs
2

Top Bar

The top bar spans the full width of the workspace and gives you quick access to project-level controls.

Model Selector

Choose which AI model powers your session. Models are grouped by provider (Anthropic, OpenAI, Google, etc.). The selector shows the current model name and lets you switch without restarting your session.

VPS Status

Shows the connection state of your linked VPS server. Green means connected and ready. If the connection drops, you'll see a reconnect option here.

Claude Code Detection

When Claude Code is running in the terminal, the top bar displays a status indicator. This helps you know when the AI is actively working on your code.

3

File Explorer

The left sidebar shows your project's file tree, loaded from your VPS via SFTP.

  • • Click a file to open it in the code editor in the LLM-Dev panel below.
  • • Folders expand/collapse with a click. The tree auto-refreshes when Claude makes changes.
  • • Status dots next to files indicate changes: green for new/modified, red for errors.
Tip: The File Explorer reads from your VPS filesystem. If you don't see recent changes, click the refresh button at the top of the tree.
4

Terminal Chat

The center panel is your main interaction point with Claude Code. This is a real terminal connected to your VPS via SSH.

How to use it

  1. 1. Type claude to start a new Claude Code session, or claude --resume to continue where you left off.
  2. 2. Describe what you want built in plain language. Claude reads your codebase, writes code, and commits changes.
  3. 3. Claude may ask yes/no questions about file edits — respond directly in the terminal.

Bubble View

The terminal includes an optional "bubble view" that renders Claude's responses in a chat-like format. Toggle between raw terminal and bubble view using the view switcher in the terminal header.

Tip: You can run any shell command here too — npm install, python app.py, git log, etc. It's a full terminal.
5

Preview Panel

The right panel is a built-in browser for previewing your web application as you build it.

Address Bar

Type any URL to load it. For local development servers (e.g., http://localhost:3000), enter the URL and see your site rendered in real time.

DevTools

Click the "Open in Browser" button to open the current URL in a new browser tab where you have access to full Chrome DevTools for debugging.

Drag Resize

Grab the left edge of the Preview Panel to drag and resize it. This lets you give more or less screen space to the preview vs. the terminal.

Tip: The docs you're reading right now are loaded in the Preview Panel! On first project open, HubLLM automatically shows this help center.
6

LLM-Dev Panel

The bottom panel is a collapsible, drag-resizable workspace with multiple tabs. Drag the top edge to expand or shrink it.

Terminals

Up to four split-pane terminals, each running an independent SSH session to your VPS. Use these for running dev servers, watching logs, or running tests in parallel with your Claude session.

Code Editor

A Monaco-powered code editor (the same engine as VS Code). Click a file in the File Explorer to open it here. Supports syntax highlighting for all major languages, and you can have multiple files open in tabs.

File Browser

An SFTP-based file browser for navigating your VPS filesystem. Upload, download, and manage files directly.

Docker & Logs

Monitor running Docker containers and view their logs, all from within the workspace.
